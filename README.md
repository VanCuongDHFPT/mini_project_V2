Kiến thức Về Rest API
Trong đây Rest API có 3 thành phần Rest, RestFul, API 
+Đầu tiên mình sẽ nói về Rest
-Rest là một bản thiết kết dùng để định nghĩa những API 
-Rest cũng giống như tờ công thức bên trong nó có cách thiết kế API 
-Sử dụng HTTP method (get , post, put, delete) 
-Resourse đc dịnh dạng bằng URL
+ RestFul
-Từ những cái bảng thiết kế của Rest thì ResFul sẽ Implement lại và tạo ra những API
-RestFul là một chuẩn dùng để tạo ra những API
-Có thể áp dụng những ngôn ngữ hoặc FrameWork nòa
